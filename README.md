## 🛠️ Repository Structure & Git Workflow

The development followed a structured branching strategy:

* `main` — Primary production branch containing the final code.
* `develop` — Development branch where completed features are merged.
* `feature/body` — Feature branch dedicated to assembling the main résumé body content.
  * `patch/content/primary` — Primary content (header, contacts, avatar, intro).
  * `patch/content/secondary` — Secondary content (skills, experience, education, languages, fun facts).
